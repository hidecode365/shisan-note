# CLAUDE.md — docs/ Vault

## このVaultの役割

このディレクトリ（`docs/`）は ShiSan Note の**要件・設計ドキュメント専用の Obsidian Vault**である。  
実装コード（TypeScript・Rust・SQL 等）はここでは扱わない。

実装規約・ビルドコマンド・プロジェクト構成は、親ディレクトリ（`shisan-note/`）の以下を参照すること：

- `CLAUDE.md`（プロジェクトルート）— 全機能共通の実装ルール・技術スタック
- `.claude/rules/backend.md` — Rust バックエンド実装ルール
- `.claude/rules/frontend.md` — React フロントエンド実装ルール

---

## このVaultの構成

### `00-overview.md`

プロダクトのコンセプト・差別化軸・ターゲットユーザー・収益化モデル・非機能要件・競合分析を記述する。  
「なぜこのプロダクトを作るか」の原点文書。

### `01-architecture.md`

技術スタック（Tauri v2・React 19・SQLite 等）の選定理由、フロントエンド／バックエンドのレイヤー構成図、データフロー、ルーティング、AI 機能のアーキテクチャを記述する。  
実装判断の根拠として参照する。

### `02-data-model.md`

SQLite の全テーブル定義（DDL）・デフォルトカテゴリの初期データ・DB ファイルパスを記述する。  
スキーマの正式定義はこのファイルが唯一の情報源。

### `adr/` — Architecture Decision Records

設計上の重要な決断とその背景・理由・トレードオフを記録するログ。  
決定済みの判断を覆す場合や背景を確認したい場合にここを参照する。

| ファイル | 内容 |
| --- | --- |
| `0001-local-first.md` | ローカルファーストアーキテクチャ（外部送信なし）の採用理由 |
| `0002-inline-entry-ux.md` | 記帳 UI をインライン表入力方式（スプレッドシート型）に変更した理由 |

### `features/FXX-*.md` — 機能仕様書

機能ごとの画面レイアウト・フィールド定義・実装上の注意・実装メモを記述する。  
実装作業前に必ず対応する機能ファイルを読むこと。

| ファイル | 内容 |
| --- | --- |
| `F01-entry.md` | 記帳機能（インライン表入力・3モード設計） |
| `F02-accounts.md` | 口座・資産管理 |
| `F10-settings.md` | 設定画面（ショートカットキー・永続化） |

**番号の割り振りについて**：現時点で存在するファイルは F01・F02 が連番で、F10 が離れた番号となっている。  
> ※未確認: `01-architecture.md` のルーティング定義には F03〜F09 に相当する画面（ダッシュボード・取引一覧・グラフ等）が存在するため、将来それらに対応する番号として予約されている可能性があるが、割り振りの意図は本人に確認していない。

## 執筆ルール

実際のファイルから読み取れるパターンのみを記す。

### ファイル命名

| 種別 | 規則 | 例 |
| --- | --- | --- |
| ADR | `NNNN-kebab-case.md`（4 桁ゼロ埋め） | `0001-local-first.md` |
| 機能仕様 | `FNN-kebab-case.md`（2 桁番号） | `F01-entry.md` |
| 概要・設計 | `NN-kebab-case.md`（2 桁番号） | `00-overview.md` |

### frontmatter

現状、全ファイルに YAML frontmatter は存在しない。すべてのファイルは `#` 見出しで始まる。

### 見出し構造

- `#`：ファイルタイトル（機能コードを含む場合は `# F01 — 記帳機能` 形式）
- `##`：主要セクション（概要・画面レイアウト・実装上の注意・実装メモ など）
- `###`：サブセクション（フィールド定義・ボタン・モード別説明 など）

### ADR の書き方

ADR 0001 と 0002 でメタデータ（ステータス・日付）の記法が統一されていない  
（0001 は `##` 見出し形式、0002 は箇条書き形式）。**現状統一されていない。**

### 機能仕様書の実装メモ欄

各機能ファイルの末尾に `## 実装メモ` セクションを設ける。  
実装中に判明した機能固有の決定事項・ハマりどころのみをここに記録する。  
全機能に共通する事項はルートの `CLAUDE.md` または `.claude/rules/` に書くこと。

## 要件変更時の注意

このVault内のドキュメントを変更した場合、**Vaultだけで完結させないこと**。  
以下のファイルにも反映されているか必ず確認する：

- `../REQUIREMENTS.md`（プロジェクトルートの要件定義書）
- `../ToDo.md`（タスク管理ファイル）

変更の大きさに関わらず、上記 2 ファイルとの整合性をセッション終了前に確認すること。
