# 00 — プロダクト概要・コンセプト・非機能要件

## コンセプト

「プライバシーファースト・ローカル完結の、モダンUIを持つ資産管理ノート」

### 核心的な差別化軸

1. **完全ローカル保存**：データはユーザーのPC内にのみ存在し、外部サーバーへの送信は一切行わない
2. **モダンUI**：既存のWindows家計簿ソフト（てきぱき家計簿マム・うきうき家計簿等）が2000年代UIから脱せていない空白地帯を狙う
3. **複数口座・資産の統合管理**：銀行・証券・仮想通貨・モバイルペイを一元管理
4. **軽量・高速**：Tauri採用によりインストーラー10MB以下・メモリ使用量30〜50MB

### ターゲットユーザー

- 金融口座をクラウドに連携することに抵抗感がある層
- セキュリティ・プライバシー意識の高いITリテラシー層
- 複数の銀行口座・証券口座・仮想通貨を持つ資産管理意識の高い層
- 既存の家計簿アプリのUIに不満を感じている層

## 非機能要件

### パフォーマンス

- 起動時間：3秒以内
- 取引一覧の表示：1万件でも200ms以内
- インストーラーサイズ：20MB以内目標

### セキュリティ

- データはローカルSQLiteに保存（将来的にAES-256暗号化）
- 外部ネットワーク通信はAI機能使用時のみ（ユーザーが明示的に設定した場合）
- アプリ内パスワードロック機能（将来）

### 対応OS

- Windows 11（優先）
- Windows 10（対応予定）
- macOS・Linux（将来・Tauriのクロスプラットフォーム機能を活用）

### アクセシビリティ

- 日本語UI（v1.0）
- ダークモード対応（shadcn/uiのテーマ機能を活用）
- フォントサイズ変更（将来）
