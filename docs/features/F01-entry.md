# F01 — 記帳機能

## 概要

日々の収支を快速に入力できる記帳画面。
「保存して次へ」ボタンで連続入力を可能にし、右ペインに取引一覧をリアルタイム表示することで、入力確認をしながら複数件を素早く記帳できる。

---

## 画面レイアウト

```text
┌─────────────────────────────────────────────────────┐
│ トップバー：「記帳」タイトル ＋ CSV取込ボタン         │
├────────────────────────────────────────────────────-┤
│ 口座ストリップ（横スクロール）                        │
│ [三菱UFJ ¥1,240,000] [楽天銀行 ¥380,000] ... [総資産]│
├──────────────────┬──────────────────────────────────┤
│ 左ペイン（50%）  │ 右ペイン（50%）                   │
│ 新規記帳フォーム │ 取引一覧（選択口座でフィルタ）     │
└──────────────────┴──────────────────────────────────┘
```

---

## 口座ストリップ

- 全口座をカード形式で横並び表示（横スクロール対応）
- 各カード：口座名・残高・種別アイコン・種別カラー
- 選択中カードをアクセントボーダーでハイライト
- 右端に「総資産」カードを固定表示
- カードクリックで右ペインの取引一覧がその口座でフィルタされる

---

## 記帳フォーム（左ペイン）

### フィールド定義

| フィールド | 種別 | 必須 | 備考 |
| -- | --- | --- | --- |
| 種別 | トグルボタン | ✓ | 支出 / 収入 / 振替 |
| 金額 | 数値入力 | ✓ | カンマ区切り表示、支出は赤・収入は緑 |
| カテゴリ | グリッド選択 | - | 4列アイコン＋ラベル形式 |
| 日付 | date picker | ✓ | デフォルト：今日 |
| 口座 | ドロップダウン | ✓ | 口座ストリップの選択と連動 |
| 支払方法 | ドロップダウン | - | 現金 / クレジットカード / Suica / PayPay / 口座引落 / その他 |
| メモ・店名 | テキスト入力 | - | |

### ボタン

- **保存して次へ**：保存後にフォームをリセット（日付・口座は維持）
- **クリア**：フォームをリセット

### 振替モード

種別「振替」選択時：

- 「口座」→「振替元口座」に変更
- 「振替先口座」フィールドを追加表示
- カテゴリは非表示

---

## 取引一覧（右ペイン）

- テキスト検索（店名・メモ）
- 月フィルタ（デフォルト：当月）
- 絞り込みボタン（種別・カテゴリ）
- 取引行：種別アイコン・店名・日付・カテゴリ・金額・口座名
- **直前に保存した行を緑ハイライト表示**（2秒後に解除）
- フッター：件数・支出合計・収入合計

---

## カテゴリグリッド

デフォルト8カテゴリをアイコン＋ラベルで4列表示。
選択中カテゴリをアクセントカラーでハイライト。

| アイコン | ラベル |
| --- | -- |
| ShoppingCart | 食費 |
| Train | 交通費 |
| Bolt | 光熱費 |
| Tv | 娯楽 |
| Home | 住居 |
| FirstAid | 医療 |
| Shirt | 衣服 |
| Dots | その他 |

## 実装上の注意

- フォームのsubmitはbuttonのonClickで処理（formタグのsubmitイベントは使わない）
- 金額入力はinputのonChangeでカンマフォーマット処理
- 保存処理はTauriコマンド `create_transaction` を呼び出す
- 保存後に取引一覧を再取得して即座に右ペインに反映する

---

## モード設計

### 3つのモード

| モード | 説明 |
| --- | --- |
| 通常モード | 一覧表示のみ。追加・編集不可 |
| 追加モード | 新規入力行を追加できる。確定済み行は変更不可 |
| 編集モード | 確定済み行をダブルクリックで編集できる。新規行の追加不可 |

### モード切り替えUI

ツールバーのレイアウト：

[財布▼] [残高] [＜][月][＞] [検索] [+ 追加 | ✏️ 編集] -------- [一括保存]

- 「+ 追加」「✏️ 編集」はトグルボタンとして並列配置（セグメントボタン風）
- アクティブなモードのボタンをハイライト表示
- 通常モード時はどちらもハイライトなし

### 追加モードの動作

- 「+ 追加」ボタン押下で追加モードへ遷移、新規入力行が展開される
- 確定済み行はダブルクリックしても編集不可
- Shift+Enter または「+ 行を追加」ボタンで次の入力行を追加
- 一括保存（Ctrl+Enter）で下書きを確定 → 通常モードへ戻る
- 「+ 追加」ボタン再押し または Esc でキャンセル
  - 未入力の場合：即キャンセル → 通常モードへ
  - 1行以上入力済みの場合：確認ダイアログ → OK で通常モードへ

### 編集モードの動作

- 「✏️ 編集」ボタン押下で編集モードへ遷移
  - 確定済み行全体が編集可能になる
  - どのセルにもカーソルは当たっていない状態で開始
- 編集モード中はセルをクリックするとフォーカスが当たり編集できる
- 変更したセルの背景色を変える（未保存の変更を視覚的に示す）
- 一括保存ボタンに変更件数を表示（例：「一括保存（3件変更）」）
- 一括保存（Ctrl+Enter）で変更を確定 → 通常モードへ戻る
- 「✏️ 編集」ボタン再押し または Esc でキャンセル
  - 未変更の場合：即キャンセル → 通常モードへ
  - 1セル以上変更済みの場合：確認ダイアログ → OK で通常モードへ（変更は破棄）

### モード排他制御

- 追加モード中は編集モードへの遷移不可（逆も同様）
- 片方のモード中にもう片方のボタンを押した場合：
  - 未変更・未入力なら即切り替え
  - 変更・入力済みなら確認ダイアログ → OK で切り替え

### 一括保存ボタンの表示

| 状態 | ボタン表示 |
| --- | --- |
| 通常モード | 一括保存（0件）・非活性 |
| 追加モード・下書きあり | 一括保存（N件追加） |
| 編集モード・変更あり | 一括保存（N件変更） |

## 実装メモ（Claude Codeが追記する欄）

実装中に判明した、この機能固有の重要な決定事項・ハマりどころをここに記録する。
（全機能に共通する事項は CLAUDE.md または .claude/rules/ に書くこと）

- カテゴリグリッドのアイコンは lucide-react のコンポーネント名を DB の `icon` カラムに文字列で保存し、フロントの `ICON_MAP` でルックアップする方式を採用。仕様の `FirstAid` は lucide-react に存在しないため `Stethoscope` に変更、`Bolt` → `Zap`、`Dots` → `MoreHorizontal`。
- `get_transactions` で `if/else` 内の `stmt` の borrow が `?` の一時変数と lifetime 競合するため、各ブランチ末尾で `rows` 変数に明示的に収集してから返すパターンに修正。
- 残高更新と取引INSERT は rusqlite の `conn.transaction()` でアトミックに処理している。削除時も同トランザクション内で残高を逆算更新する。
- Tauri コマンドのパラメータ名は JS 側も snake_case で渡す必要がある（`account_id`, `year_month`）。`src/lib/tauri.ts` のラッパーで変換している。
- **ADR-0002対応（インライン表入力方式）**：左右分割を廃止し1枚のテーブルに統合。下書き行は `DraftRow[]` でReact stateのみ管理（DB保存なし）。一括保存は `create_transactions_bulk` でRust側 `conn.transaction()` によりアトミック処理。ショートカット（Shift+Enter/Ctrl+Enter）は `src/lib/shortcuts.ts` の `ShortcutDef` 型で管理し、DBの `settings` テーブルから起動時にロード、将来F10設定画面から変更可能な構造にしてある。グローバルキーハンドラは `useRef` パターンで最新クロージャを保持し、stale closure を回避している。
