# F10 — 設定画面

## 概要

アプリ全体の設定を管理する独立画面。
左サイドバーでカテゴリを選択し、右エリアに設定項目を表示する2ペイン構成。
設定値はSQLiteのsettingsテーブルまたは設定ファイルに保存し、起動時に読み込む。

---

## 画面レイアウト

```text
┌─────────────────────────────────────┐
│ ← 戻る   設定                        │  ← トップバー
├────────────┬────────────────────────┤
│ カテゴリ   │ 設定項目エリア            │
│ ─────────  │                         │
│ 一般       │                         │
│ 記帳       │                         │
│ 表示       │                         │
│ （将来追加）│                         │
└────────────┴────────────────────────┘
```

- 左サイドバー幅：固定（約180px）
- 右エリア：残り全幅
- 戻るボタンで前の画面に戻る

## カテゴリ一覧

### 一般
- アプリのバージョン表示（読み取り専用）
- 将来：言語設定・テーマ（ライト/ダーク）

### 記帳
- ショートカットキー設定（詳細は後述）
- 将来：デフォルト口座・デフォルト区分

### 表示
- 将来：通貨表示形式・日付フォーマット

## ショートカットキー設定（記帳カテゴリ）

### UI構成

各ショートカットは以下のUIで設定する：
[□ Ctrl] [□ Alt] [□ Shift] [□ Win]  [プルダウン ▼]  →  表示例：Ctrl+Enter

- 修飾キー（Ctrl / Alt / Shift / Win）：チェックボックスで複数選択可
- メインキー：プルダウンで選択
- 選択内容はリアルタイムで「Ctrl+Enter」形式でプレビュー表示

### プルダウンの選択肢

Space / Tab / Enter / F1〜F12 / A〜Z / 0〜9

### 設定項目

| 設定名 | 説明 | デフォルト |
| --- | --- | --- |
| 次の入力行を追加 | 記帳入力中に次の行を追加する | Shift+Enter |
| 下書きを一括保存 | 記帳入力中の下書きをDBに確定保存する | Ctrl+Enter |

### 保存・リセット

- [保存]ボタンで設定をDBに書き込む
- [デフォルトに戻す]ボタンでデフォルト値にリセット

---

## 設定の永続化

- 設定値はSQLiteの `settings` テーブルに key-value 形式で保存
- テーブル定義：`settings(key TEXT PRIMARY KEY, value TEXT, updated_at TEXT)`
- 起動時に全設定を読み込み、アプリ全体で参照できるようにする
- ショートカットキーのkey名：`shortcut.entry.add_row` / `shortcut.entry.save_draft`

---

## 実装上の注意

- 設定値の読み込みはアプリ起動時に1回実行し、Reactのコンテキストで管理する
- ショートカットのキー定義はハードコードせず、設定値から動的に参照すること
- 将来のカテゴリ追加を想定し、左サイドバーは配列から動的に生成する
